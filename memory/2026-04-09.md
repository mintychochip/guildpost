## Root Cause Found
The frontend JavaScript in `index.astro` and `minecraft.astro` references `createClient` from `@supabase/supabase-js`, but:

1. The library is **never loaded** (no CDN script tag, no bundled import)
2. The scripts use `is:inline` which prevents Astro from processing the ES6 import
3. This causes `createClient` to be undefined → Supabase calls fail silently

## Evidence
- Direct API call with anon key works: `curl .../rest/v1/servers` returns data
- CORS is properly configured (Access-Control-Allow-Origin: https://guildpost.tech)
- Anon key is valid (tested successfully)
- No Supabase script tag in Layout.astro

## Fix Required
Add Supabase library to the layout - either:
1. CDN: `<script src="https://cdn.jsdelivr.net/npm/@supabase/supabase-js@2/dist/umd/supabase.min.js"></script>`
2. Or remove `is:inline` from scripts so Astro bundles the import
